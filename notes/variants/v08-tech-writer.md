## 内容创作工作流

每篇文章遵循这个流程：

```
选题（30min）→ 大纲（30min）→ 初稿（2-3h）→ AI 润色（30min）→ 配图排版（30min）→ 发布 → 推广
```

### 选题策略

| 类型 | 占比 | 示例 | 难度 |
|------|------|------|------|
| 教程型 | 40% | 《从零用 Next.js 搭建 AI 聊天应用》 | 中 |
| 踩坑型 | 25% | 《Drizzle ORM 迁移踩的 5 个坑》 | 低 |
| 原理型 | 20% | 《RAG 到底是怎么工作的？图解》 | 高 |
| 观点型 | 15% | 《AI 辅助编程一个月，我的效率真的提升了吗？》 | 低 |

### AI 辅助写作流程

1. **选题**：Claude 头脑风暴 10 个选题，挑最有价值的
2. **大纲**：自己写骨架，Claude 补充遗漏
3. **初稿**：核心技术细节自己写（这是学习过程）
4. **润色**：Claude 改语法、优化结构、补过渡句
5. **配图**：Excalidraw 画架构图，CleanShot 截操作步骤
6. **多平台**：博客完整版，掘金精简版，Twitter 核心观点

**文章质量标准**：有代码必须能跑（附 GitHub 链接），有架构必须有图，有观点必须有数据支撑，标题要有搜索价值。

## 增长飞轮

```
写文章 → 涨粉丝 → 被发现 → 拿机会
  ↑                              ↓
做项目 ← 学技术 ← 教别人 ← 写文章
```

费曼学习法闭环：做项目遇到问题 → 深入研究 → 写文章发现理解不够 → 继续深入 → 读者提问发现盲区 → 再次深入。每一轮理解更深一层。

---

> 核心理念：不是"学完再写"，而是"边学边写"。写作不是学习之后的总结，写作本身就是学习。3 个月后，你的 30 篇文章就是最好的简历，你的粉丝就是最强的背书。面试官问你什么，你都能说"我写过"。
